# 코딩 표준

이 디렉토리는 언어별 코딩 표준과 관련 설정 파일을 정리한다.

코딩 표준은 개인의 취향보다 프로젝트 전체의 일관성과 코드 품질을 유지하기 위한 기준이다.  
코드의 의도, 책임, 오류 처리 방식, 자료형의 의미, 소유권과 수명처럼 유지보수에 중요한 정보는 일관된 방식으로 드러나야 한다.

이 저장소의 코딩 표준은 널리 검증된 여러 코딩 표준을 참고하되 실제 개발 환경에서의 가독성과 유지보수성을 고려하여 일부 항목을 현실적으로 조정하였다.

팀이나 회사에 이미 정해진 코딩 표준이 있다면 해당 표준을 우선한다.  
이 코딩 표준은 개인 학습, 개인 프로젝트 또는 코딩 표준을 직접 정할 수 있는 프로젝트에서 사용할 수 있는 기준이다.

좋은 코딩 표준을 기준으로 코드를 작성하면 일관된 코드 구조와 명확한 이름, 안정적인 오류 처리, 읽기 쉬운 API 설계를 훈련할 수 있다.  
이 과정은 가독성이 높고 유지보수하기 쉬운 코드를 작성하는 습관을 기르는 데 도움이 된다.

## 기본 방향

- 코드는 실행 결과뿐 아니라 작성자의 의도까지 드러내야 한다.
- 이름, 자료형, 함수 시그니처, API 계약으로 의미를 먼저 표현한다.
- 주석은 코드만으로 드러나지 않는 이유, 제약, 특별한 판단을 설명할 때 사용한다.
- 실패 조건은 숨기지 않고 가능한 한 빠르고 분명하게 처리한다.
- 자동 포맷터로 맞출 수 있는 규칙은 도구에 맡기고 의미를 다루는 규칙은 코드 리뷰와 정적 분석 도구로 확인한다.
- 언어마다 다른 위험 요소와 관용은 해당 언어의 특성에 맞게 다룬다.

## 사용 기준

- 새 프로젝트를 시작할 때는 사용하는 언어의 코딩 표준을 먼저 확인한다.
- 프로젝트의 도메인, 성능 요구 사항, 플랫폼 제약에 따라 필요한 항목은 명확히 조정한다.
- 자동 포맷터는 코드 레이아웃을 맞추기 위한 도구로 사용한다.
